# 고구마마켓 프로젝트 가이드

## 프로젝트 개요

개발 연습용 중고거래 웹사이트입니다.

- **Next.js** — 웹페이지를 만드는 프레임워크 (React 기반)
- **Supabase** — 데이터베이스 + 로그인 기능을 제공하는 서비스
- **Vercel** — 만든 웹사이트를 인터넷에 올려주는 배포 서비스
- **실제 앱 코드 위치**: `goguma-market/` 폴더 안에 있음

---

## Claude 작업 지침

### 커뮤니케이션

- 개발 용어는 비개발자도 이해할 수 있게 쉽게 설명한다.
  - 예시: "컴포넌트" → "화면 조각", "라우팅" → "페이지 이동", "API" → "데이터를 주고받는 통로", "스키마" → "데이터 저장 구조"
- 단계마다 "지금 ~을 하고 있어요" 형태로 진행 상황을 자주 알린다.
- 작업이 끝나면 한두 줄로 무엇이 바뀌었는지 요약한다.

### 작업 방식

- 사용자가 지시한 것만 단계적으로 수행한다. 요청하지 않은 기능 추가, 코드 정리, 구조 변경은 하지 않는다.
- 할 수 있는 건 최대한 직접 작업한다.
- 꼭 사람이 직접 해야 하는 것(예: Supabase 대시보드 설정, 환경 변수 입력, Vercel 배포 설정)만 알기 쉽게 번호를 붙여 단계별로 안내한다.
- 개발이 끝나면 항상 미리보기(preview)를 열어서 바로 확인할 수 있게 한다.

### 코드 스타일

- 주석은 꼭 필요한 경우에만 한국어로 작성한다.
- 불필요한 코드, 미래를 위한 추상화, 사용하지 않는 변수는 추가하지 않는다.
- 보안에 취약한 코드(SQL 인젝션, XSS 등)는 절대 작성하지 않는다.

### Next.js 주의사항

- `node_modules/next/dist/docs/` 안의 가이드를 참고한다. 이 프로젝트의 Next.js 버전이 일반적인 예시와 다를 수 있다.
- App Router 방식(`app/` 폴더)을 사용한다.

### Supabase 주의사항

- 데이터 저장 구조를 바꾸기 전에 반드시 `list_tables`로 현재 구조를 먼저 확인한다.
- 문제가 생기면 `get_logs`와 `get_advisors`로 원인을 먼저 파악한다.

### Vercel 배포

- Vercel 프로젝트 ID: `prj_loBPTFqovVkVqM24Nn8PNcweyCxs`
- 배포는 `goguma-market/` 폴더 기준으로 이루어진다.
- 환경 변수(Supabase 주소, API 키 등)는 Vercel 대시보드에서 직접 입력해야 한다.
